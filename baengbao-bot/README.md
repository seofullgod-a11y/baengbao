# แบ่งเบา — LINE Bot (เฟส 1)

บอทไลน์ผู้ช่วยบัญชีร้านอาหาร: พิมพ์รายรับ-รายจ่าย หรือถ่ายรูปบิล แล้ว Claude แปลงเป็นรายการบัญชี เก็บลง PostgreSQL และตอบยอดสรุปรายวันกลับทันที

## สิ่งที่บอททำได้ตอนนี้

- พิมพ์ภาษาไทยธรรมชาติ → ลงบัญชีให้ (เช่น `ขายกะเพรา 5 จาน 250`, `ซื้อหมู 800`, `จ่ายค่าแก๊ส 450`)
- ถ่ายรูปบิล/ใบเสร็จ → Claude อ่านยอด แยกรายการ ลงบัญชีให้
- คำสั่ง `สรุป` / `วันนี้` → ยอดวันนี้ • `เดือนนี้` → ยอดทั้งเดือน
- ตอบกลับด้วย reply message (ไม่กินโควตา push ของ LINE = ฟรี)

## โครงสร้าง

```
baengbao-bot/
├── server.js      ← webhook, verify signature, รับ event, ตอบกลับ
├── ai.js          ← เรียก Claude แปลงข้อความ/รูปบิล เป็น JSON
├── db.js          ← PostgreSQL: schema + query helpers
├── package.json
├── .env.example
└── README.md
```

## Environment variables ที่ต้องตั้ง

| ตัวแปร | เอามาจาก |
|---|---|
| `LINE_CHANNEL_SECRET` | LINE Developers Console → channel → Basic settings |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE Developers Console → Messaging API → Issue token (long-lived) |
| `ANTHROPIC_API_KEY` | console.anthropic.com |
| `DATABASE_URL` | Railway ใส่ให้อัตโนมัติเมื่อเพิ่ม Postgres |

## Deploy ขึ้น Railway

1. push โค้ดนี้ขึ้น GitHub repo
2. Railway → New Project → Deploy from GitHub repo → เลือก repo
3. ในโปรเจกต์เดียวกัน กด **+ New → Database → PostgreSQL** (Railway จะ inject `DATABASE_URL` ให้บริการ web อัตโนมัติ ถ้าไม่ได้ ให้ไปที่ Variables ของ service แล้ว reference ตัวแปรจาก Postgres)
4. ไปที่ service ของบอท → **Variables** → ใส่ `LINE_CHANNEL_SECRET`, `LINE_CHANNEL_ACCESS_TOKEN`, `ANTHROPIC_API_KEY`
5. Railway รัน `npm start` ให้เอง (ไม่ต้องมี `railway.json`) — schema ตารางสร้างเองตอน start
6. เปิด **Settings → Networking → Generate Domain** เพื่อได้ public URL เช่น `https://baengbao-bot-production.up.railway.app`

> ลองเปิด `https://<your-domain>/health` ควรเห็น `{"ok":true}` ก่อนไปต่อ

## ตั้งค่า Webhook ฝั่ง LINE (ทำหลัง deploy)

**ใน LINE Developers Console** (developers.line.biz/console → channel → แท็บ Messaging API):
1. **Webhook URL** = `https://<your-domain>/webhook` → Update
2. เปิด **Use webhook** = ON
3. กด **Verify** → ควรขึ้น Success (เป็น POST ว่างๆ บอทตอบ 200)

**ใน LINE Official Account Manager** (manager.line.biz → ตั้งค่า → การตอบกลับ / Response settings):
1. ปิด **การตอบกลับอัตโนมัติ (Auto-reply)** — ไม่งั้นมันจะแย่งตอบทับบอท
2. ปิด **ข้อความทักทายเพื่อนใหม่ (Greeting)** ก็ได้ (บอทมี follow event ทักเองอยู่แล้ว)
3. **โหมดการตอบกลับ / Chat** ตั้งให้ใช้ Webhook/Bot ได้ (ถ้ามีตัวเลือก ให้เปิดเป็น Bot)

## ทดสอบ

แอดเพื่อน OA "แบ่งเบา" แล้วพิมพ์ `ขายกะเพรา 5 จาน 250` ควรได้:

```
✅ บันทึกแล้ว
รายรับ +250 ฿ (กะเพรา x5)
━━━━━━━
วันนี้: รายรับ 250 / รายจ่าย 0
กำไรสุทธิ +250 ฿
```

## รันในเครื่อง (ออปชัน)

```bash
cp .env.example .env   # แล้วเติมค่าจริง
npm install
npm start
```
local ต้องเปิด tunnel (เช่น ngrok/cloudflared) เพื่อให้ LINE ยิง webhook เข้ามาได้ — ขึ้น Railway ตรงๆ ง่ายกว่า

## ต้นทุนคร่าวๆ

- LINE: reply message **ไม่กินโควตา** — ฟรี ส่วน push (สรุปอัตโนมัติประจำวัน ถ้าทำในเฟสถัดไป) ใช้โควตา 200/เดือนของ free plan
- Claude: ต่อ 1 ข้อความใช้ haiku (ถูกมาก) / รูปบิลใช้ sonnet เปลี่ยนรุ่นได้ที่ env `TEXT_MODEL`/`VISION_MODEL`

## เฟสถัดไป (ยังไม่รวมในนี้)

- เมนู + สูตรต้นทุน → กำไรต่อจาน (LIFF)
- Dashboard กราฟรายวัน/รายเดือน (LINE Login)
- ดึงยอดเดลิเวอรี่ (forward อีเมล payout / อ่านรูปสรุปยอด)

# เฟส 2 — เมนู + กำไรต่อจาน (LIFF mini-app)

เพิ่มหน้าเว็บจัดการเมนูที่เปิดในไลน์ (LIFF) ผูกกับ userId เดียวกับบอท ตั้งราคาขาย/ต้นทุน แล้วเห็นกำไรต่อจาน + มาร์จิน พร้อมสไลเดอร์ "วัตถุดิบขึ้นราคา %" และบอทจะโชว์กำไรให้ตอนบันทึกการขายที่ตรงกับเมนู

## ของใหม่ในเฟสนี้
- ตาราง `menus` (สร้างเองตอน start)
- หน้า mini-app: `https://<bot-domain>/app`
- API: `GET/POST/PUT/DELETE /api/menus` (ยืนยันตัวตนด้วย LIFF access token)
- คำสั่งแชทใหม่: `เมนู` (ลิงก์เปิด mini-app), `กำไรเมนู` (ดูกำไรต่อจานทุกเมนู)
- ตอนพิมพ์ขายเมนูที่ตั้งไว้ บอทจะเพิ่มบรรทัด `💰 กำไรเมนูนี้ ~X ฿`

## ตั้งค่า LINE Login + LIFF (ทำครั้งเดียว)

1. LINE Developers Console → **Provider เดิม (baengbao)** → **Create a new channel** → เลือก **LINE Login**
2. กรอกชื่อ (เช่น "แบ่งเบา") + ข้อมูลที่จำเป็น → สร้าง
3. เข้า channel LINE Login นั้น → แท็บ **LIFF** → **Add**
   - **Endpoint URL** = `https://<bot-domain>/app`
   - **Size** = Full
   - **Scope** = ติ๊ก `profile` และ `openid`
   - กด Add
4. คัดลอก **LIFF ID** (หน้าตาเช่น `1657812345-AbcdWxyz`)
5. Railway → service บอท → **Variables** → เพิ่ม `LIFF_ID` = ค่าที่คัดลอก → Railway redeploy ให้เอง

## ทดสอบ
- ในแชทแบ่งเบา พิมพ์ `เมนู` → กดลิงก์ที่บอทส่ง → หน้า mini-app เปิดในไลน์
- เพิ่มเมนู เช่น ข้าวกะเพรา ขาย 60 / วัตถุดิบ 22 / ค่าแรง 14 → เห็นกำไร 24 ฿/จาน
- กลับมาแชท พิมพ์ `ขายกะเพรา 5 จาน 250` → บอทเพิ่มบรรทัดกำไรเมนูให้
- พิมพ์ `กำไรเมนู` → ดูกำไรต่อจานทุกเมนูเรียงจากมากไปน้อย

## env เพิ่ม
| ตัวแปร | ค่า |
|---|---|
| `LIFF_ID` | LIFF ID จาก LINE Login channel |

> หมายเหตุ: ตอนพัฒนา เจ้าของ channel เปิด LIFF ได้เลย ถ้าจะให้ลูกค้าคนอื่นเปิดได้ ต้อง Publish channel LINE Login ก่อน

# เฟส 3 — Dashboard + กราฟ + แก้/ลบรายการ

เพิ่มแท็บ **รายงาน** ในหน้า mini-app เดิม (ไม่ต้องตั้งค่า LINE เพิ่ม — ใช้ LIFF ID เดิม)

## ของใหม่
- หน้า mini-app เป็นแบบ 2 แท็บ: **เมนู** | **รายงาน**
- แดชบอร์ด: การ์ดรายรับ/รายจ่าย/กำไรสุทธิของเดือน, กราฟแท่งรายรับ-รายจ่ายรายวัน, โดนัทรายจ่ายแยกหมวด, เลื่อนดูย้อนเดือนได้
- รายการล่าสุด + ปุ่มลบ (แก้ของที่จดผิด)
- คำสั่งแชทใหม่: `รายงาน` (เปิดแดชบอร์ด)
- กราฟใช้ Chart.js (โหลดจาก CDN ตอนเปิดหน้า)

## API เพิ่ม (ยืนยันด้วย LIFF token)
- `GET /api/stats?month=YYYY-MM` — ยอดรายวัน, แยกหมวด, สรุปเดือน
- `GET /api/transactions?limit=20` — รายการล่าสุด
- `DELETE /api/transactions/:id` — ลบรายการ

## Deploy
push โค้ดทับ repo `baengbao-bot` เดิม → Railway redeploy เอง → ไม่ต้องแก้อะไรใน LINE
ทดสอบ: พิมพ์ `รายงาน` ในแชท → กดลิงก์ → สลับแท็บ "รายงาน" ดูกราฟได้เลย

# เฟส 4 — เดลิเวอรี่ (อ่านหน้าสรุปยอด)

แคปหน้าสรุปยอด/รายได้จาก Grab / LineMan / Shopee Food ส่งเข้าแชท → AI แยกว่าเป็น "หน้าสรุปเดลิเวอรี่" (ไม่ใช่บิลซื้อของ) แล้วลงบัญชีให้อัตโนมัติ:
- ลง **รายรับ = ยอดขายรวม** (category `ขายเดลิเวอรี่·{platform}`)
- ลง **รายจ่าย = ค่า GP/คอมมิชชั่น** ที่ถูกหัก (category `ค่า GP·{platform}`)
- ตอบกลับโชว์ ยอดขาย / ค่า GP / ยอดสุทธิ + สรุปวันนั้น

ทำให้กราฟรายจ่ายแยกหมวดเห็นค่า GP ของแต่ละเจ้าได้ และกำไรสุทธิตรงความจริง

## ทำงานยังไง
- ไม่มี API ทางการของแพลตฟอร์มให้ดึงยอดส่วนตัว → ใช้วิธีถ่าย/แคปหน้าสรุปแล้วให้ Claude vision อ่าน (วิธีที่ทำได้จริงและถูกกฎ)
- ถ้าหน้าสรุปมีแต่ยอดโอนสุทธิ (ไม่เห็นยอดรวม/GP) → ลงเป็นรายรับสุทธิอย่างเดียว
- รองรับวันที่ในหน้าสรุป (ถ้าอ่านได้) ไม่งั้นใช้วันที่ปัจจุบัน

## Deploy
push ทับ repo เดิม → redeploy → ไม่ต้องแก้อะไรใน LINE
ทดสอบ: แคปหน้าสรุปยอด Grab/LineMan ส่งเข้าแชท → ดูว่าลงรายรับ + ค่า GP ถูกไหม → เปิด `รายงาน` ดูว่าโดนัทมีหมวด "ค่า GP" โผล่

# เฟส 5 — เปิดให้คนอื่นใช้

## ของใหม่ในโค้ด
- **Rate limit**: จำกัดการเรียก AI ต่อผู้ใช้ต่อวัน (ค่าเริ่มต้น 80 ครั้ง/คน/วัน) เก็บใน DB ตาราง `usage_daily` ทนรีสตาร์ท — กันค่า Claude API บานปลายเวลามีคนใช้เยอะ คำสั่งดูข้อมูล (สรุป/รายงาน/เมนู) ไม่นับโควตา ปรับเพดานได้ที่ env `DAILY_AI_LIMIT`
- **Onboarding**: ข้อความต้อนรับอุ่น ๆ + ตัวอย่างให้ลองพิมพ์ทันทีตอนแอดเพื่อน

## env เพิ่ม (ออปชัน)
| ตัวแปร | ค่า |
|---|---|
| `DAILY_AI_LIMIT` | เพดานเรียก AI/คน/วัน (ดีฟอลต์ 80) |

## ตั้งค่าฝั่ง LINE เพื่อเปิดสาธารณะ
1. **Publish LINE Login channel** (อันที่มี LIFF) — Developers Console → channel LINE Login → Basic settings → เปลี่ยนสถานะจาก Developing เป็น **Published** ไม่งั้นคนอื่นเปิด mini-app ไม่ได้
2. **เปิด OA ให้ค้นเจอ** — OA Manager → ตั้งค่า → ทำให้บัญชีค้นหาเจอ/แชร์ลิงก์เพิ่มเพื่อนได้
3. (แนะนำ) ใส่ Privacy Policy URL + Terms ในแต่ละ channel ก่อนเปิดจริง

## ปุ่มแอดไลน์ในหน้า landing
อัปเดตในโปรเจกต์ `baengbao-landing` แล้ว — ปุ่มสีเขียวทั้ง 3 จุดชี้ไป `https://line.me/R/ti/p/@952dxvdb`
> ตรวจสอบ ID ให้ตรงกับ OA จริงของคุณอีกที (ดูได้ที่ OA Manager) ถ้า basic ID ไม่ใช่ @952dxvdb ให้แก้ใน index.html

## ข้อควรระวังต้นทุนเมื่อเปิดจริง
- ทุกการ "จด" 1 ครั้ง = เรียก Claude 1 ครั้ง (ข้อความใช้ haiku ถูก / รูปใช้ sonnet)
- คอยดู usage ใน console.anthropic.com และตั้ง budget alert ไว้
- ถ้าคนใช้เยอะมาก ค่อยพิจารณาให้ผู้ใช้ใส่ API key ของตัวเอง (BYOK) หรือเก็บค่าบริการ

# เฟส 6 — Flex Message (การ์ดสวย)

เปลี่ยนข้อความตอบกลับสำคัญ ๆ จากข้อความธรรมดาเป็นการ์ด Flex สวย ๆ เข้าแบรนด์ (ขาว-น้ำเงิน-เขียว):
- การ์ดบันทึกรายรับ/รายจ่าย (โชว์ยอด + กำไรเมนู + สรุปวันนี้ + ปุ่มดูรายงาน)
- การ์ดสรุปวันนี้ / เดือนนี้ / รายงาน (+ ปุ่มเปิดแดชบอร์ด)
- การ์ดกำไรต่อจาน (ลิสต์เมนูเรียงตามกำไร)
- การ์ดเดลิเวอรี่ (ยอดขาย / ค่า GP / สุทธิ)
- การ์ดเปิดเมนู (+ ปุ่มจัดการเมนู)

ตัวสร้างการ์ดอยู่ในไฟล์ใหม่ `flex.js` ข้อความ error/ช่วยเหลือยังเป็นข้อความธรรมดา (เบาและชัดเจน)
ปุ่มในการ์ดลิงก์ไป LIFF dashboard อัตโนมัติเมื่อมี LIFF_ID

# เฟส 7 — แจ้งเตือนสรุปรายวันอัตโนมัติ

ทุกเย็น (ค่าเริ่มต้น 21:00 เวลาไทย) บอท push การ์ดสรุปยอดวันนั้นให้ พร้อมเทียบกำไรกับเมื่อวาน (▲/▼)

## รายละเอียด
- ส่งเฉพาะผู้ใช้ที่ "มีการจดในวันนั้น" — ประหยัด quota และไม่กวนคนที่ไม่ได้ใช้
- ผู้ใช้ปิด/เปิดเองได้: พิมพ์ "ปิดสรุป" หรือ "เปิดสรุป"
- กันส่งซ้ำตอน redeploy ด้วยการจดสถานะ last_daily_push ในตาราง bot_state
- ตัวจับเวลาเช็กทุก 1 นาที ส่งครั้งเดียวต่อวันเมื่อถึงชั่วโมงที่ตั้งไว้

## env (ออปชัน)
| ตัวแปร | ค่า |
|---|---|
| `DAILY_SUMMARY_ENABLED` | `true`/`false` (ดีฟอลต์ true) |
| `DAILY_SUMMARY_HOUR` | ชั่วโมงเวลาไทยที่จะส่ง (0–23, ดีฟอลต์ 21) |

## ⚠️ เรื่อง quota การ push
ข้อความ push นับใน quota ของ LINE OA (ต่างจาก reply ที่ฟรีไม่จำกัด) บัญชีที่ยังไม่ verify มีโควตา push ต่อเดือนจำกัด ถ้าผู้ใช้เยอะมากควรดูโควตาใน OA Manager และพิจารณา verify บัญชี หรือปิดฟีเจอร์นี้ด้วย `DAILY_SUMMARY_ENABLED=false`

## DB ที่เพิ่ม
- คอลัมน์ `users.daily_summary` (เปิด/ปิดรายคน)
- ตาราง `bot_state` (เก็บสถานะระบบ เช่น วันที่ push ล่าสุด)

# เฟส 8 — เป้ายอดขาย + แจ้งเตือนถึงเป้า

ตั้งเป้ายอดขายต่อวัน/ต่อเดือน ดูความคืบหน้าแบบ progress bar และบอทเชียร์ทันทีที่ยอดแตะเป้า

## คำสั่ง
- "เป้าวันละ 5000" — ตั้งเป้ายอดขายต่อวัน
- "เป้าเดือนละ 120000" — ตั้งเป้ายอดขายต่อเดือน
- "เป้า" — ดูความคืบหน้า (การ์ด progress bar วัน + เดือน)
